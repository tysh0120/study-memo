### マーク

大文字の場合バッファを超えてグローバルで有効
##### ' {mark}

##### \`{mark}

##### 自動マーク
##### ''

##### '.

##### '[

##### ']
直前の変更またはヤンク箇所の末尾
##### '<

##### '>
直前の選択範囲末尾

##### i + モーション: 
##### iw
カーソルを含む単語内部（記号で分割される）

##### a + モーション:
##### aw

##### aW
カーソルを含む単語内部。後ろのスペースも含まれる（記号も単語に含まれる）
##### as
カーソルを含む文章全体
##### ap
カーソルを含むパラグラフ全体
##### a)

　  da]d    [ ... ] を削除

##### at

# コマンドラインモード
##### :! {cmd}
cmdを実行する
"%" でバッファの内容を指定可能
例
```
:! ruby %
```

##### : read  !{cmd}
cmdの実行結果を現在のバッファに挿入

##### :write !{cmd}
現在のバッファの各行をcmdで実行

:! {cmd} %  と同じ？

##### :range!{フィルタ}
開始行から終了行までをフィルタ結果で置き換え
フィルタとは、標準入力で受け取った内容を変更して標準出力に出力するコマンド一般(sortなど)
例)
```
2,$!sort          # ２行目から最終行をソートして置き換え
```

### range 指定
簡単のため例は p （表示）で表記

### 置換コマンド :substitute
:{range}s/{pattern}/{置換後}
オプション指定
:{range}s/{pattern}/{置換後}/g   # range全体に対して置換
:{range}s/{pattern}/{置換後}/gc  # 一つずつ確認(confirm)
```
:%s/tst/test/gc
```

##### tips
{pattern}を指定しないと前回検索したパターンが使用される。
検索と置換を分けて2回で実行すれば、検索でハイライトを確認しながらパターンを組み立てたのち置換ができる

###### 値渡し

# パターン
/{pattern}
{pattern}: 正規表現を指定する。(少し独特)
- "(", ")" は普通に指定するとパターンの文字列と解釈されてしまうため、グループの意味を持たせるにはエスケープ必要 \\(, \\)
- "{" は繰り返しの意味を持たせるにはエスケープ必要 \\{  ただし閉じかっこはエスケープ不要
- \"[", \"]" はエスケープ不要
##### very magic検索
検索パターンの先頭に \\v を指定すると、perl的な指定の仕方ができるようになる。
##### リテラルスイッチ
\\V リテラルスイッチを先頭に指定すると、特殊文字をリテラルでマッチするような動きになる。
##### 大文字、小文字の区別
検索パターンに \\c を含めると大文字小文字区別しない。 \\Cを含めると大文字小文字区別
##### smartcase

:set smartcaseをオンにすると、検索パターンに大文字を含むときのみ大文字、小文字区別。すべて小文字の場合は区別しない。
ignorecaseと同時に指定する。

指定例)
:set ignorecase smartcase


### 検索
##### 順方向:  / {pattern}
##### 逆方向: ? {pattern}

patternを省略すると、前回使用したパターンを適用
→ n / N の方向を修正するのに使える！！
##### パターンの出現回数 %s///gn
置換コマンド%sにnオプションをつけると置換をしなくなる。空のパターンを指定することで前回使用したパターンを適用。
##### オフセット検索  / {pattern} /e
マッチしたパターンの末尾にカーソルが来る。
**使用例)**
  langs  を  languages に置換したい場合、  /lang/e  でlangの末尾にカーソルを置いて  **a**uage でsの前に追記
